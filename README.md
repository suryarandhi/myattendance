## Step-by-Step Instructions to Deploy on GitHub Pages

### **Step 1: Create a GitHub Account** (If you don't have one)
1. Go to https://github.com
2. Click "Sign up"
3. Enter your email, create a password, and choose a username
4. Verify your email address

---

### **Step 2: Create a New Repository**
1. Log in to GitHub
2. Click the **"+"** icon in the top right corner
3. Select **"New repository"**
4. Fill in the details:
   - **Repository name**: `attendance-tracker` (or any name you like)
   - **Description**: "My personal attendance tracker"
   - **Public** (select this - required for free GitHub Pages)
   - ‚úÖ Check "Add a README file"
5. Click **"Create repository"**

---

### **Step 3: Upload Your Files**
1. In your new repository, click **"Add file"** ‚Üí **"Upload files"**
2. Drag and drop these 4 files:
   - `index.html`
   - `manifest.json`
   - `icon-192.png`
   - `icon-512.png`
3. Scroll down and click **"Commit changes"**

---

### **Step 4: Enable GitHub Pages**
1. In your repository, click **"Settings"** (top menu)
2. Scroll down and click **"Pages"** (left sidebar)
3. Under "Source", select:
   - Branch: **main** (or **master**)
   - Folder: **/ (root)**
4. Click **"Save"**
5. Wait 1-2 minutes for deployment

---

### **Step 5: Access Your App! üéâ**
Your app will be live at:
```
https://YOUR-USERNAME.github.io/attendance-tracker/
```

Replace `YOUR-USERNAME` with your actual GitHub username!

## **Troubleshooting:**

**Problem:** Page not loading
- Solution: Wait 2-3 minutes after enabling Pages

**Problem:** Can't find the link
- Solution: Go to Settings ‚Üí Pages to see your link

**Problem:** Files not uploading
- Solution: Make sure files are less than 25MB each
